Privacy First Policy

Last updated: July 2026

The Kenz Core Privacy Principle: We do not want, collect, or store your data. Period. Kenz is built from the ground up as a standalone local utility.

1. Zero Data Collection & Processing

The Kenz application operates with absolute cryptographic and network isolation. All file mapping, directory structure scanning, and asynchronous Zstandard (ZSTD) multi-threaded compression pipelines run strictly on your local computer hardware.

No file content, directory names, or compression metadata are ever transferred, uploaded, or exposed to external servers or remote endpoints.

2. Local Infrastructure Architecture

Any operational records, statistics computation data, or license tokens required to operate the application are read and written using local files stored securely inside your operating system's native directory layer. No external telemetry hooks or analytics scripts are packaged inside the binary.

3. Website Infrastructure

This landing website (kenz.solutions) is a static deployment infrastructure hosted cleanly via GitHub Pages. No cookies are stored by Kenz, and no client-side tracking configurations are loaded during your session.

4. Complete Machine Sovereignty

Because your environment remains 100% offline-first, you maintain exclusive control over your datasets, codebases, and archiving pipelines. You do not need a remote cloud account to purchase, deploy, or run Kenz.

© 2026 Kenz. All processing stays local.
